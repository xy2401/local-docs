字符和字符串API

的Character类封装了char数据类型。对于J2SE第5版， Character类以支持补充字符。该API分为两类：在char代码点值以及验证代码点或地图代码点有效性的方法。

本节介绍了Character类。有关可用API的完整列表，请参见Character类规范。

字符类中的验证和映射方法

某些以前使用char基本数据类型，例如isLowerCase(char)和isDigit(char) ，由支持补充字符的方法代替，例如isLowerCase(int)和isDigit(int) 。支持以前的方法，但不能与补充字符一起使用。要创建全局应用程序并确保您的代码可与任何语言无缝配合，建议您使用这些方法的较新形式。

请注意，出于性能原因，大多数接受代码点的方法都不会验证代码点参数的有效性。您可以使用isValidCodePoint为此目的的方法。
