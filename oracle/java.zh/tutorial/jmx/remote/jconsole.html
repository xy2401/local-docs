通过JConsole公开用于远程管理的资源

如果使用现成的远程管理代理和现有的监视和管理工具（例如JConsole），则使用JMX API公开Java应用程序以进行远程管理非常简单。

要公开您的应用程序以进行远程管理，您需要使用正确的属性启动它。本示例说明了如何公开Main用于远程管理的JMX代理。

安全考虑： 

为了简单起见，在此示例中禁用了身份验证和加密安全机制。但是，在实际环境中实施远程管理时，应实施这些安全机制。 接下来是什么？ 提供指向其他JMX技术文档的指针，这些文档显示了如何激活安全性。

此示例需要Java SE平台的版本6。监视Main远程JMX代理，请按照下列步骤操作：

  1. 如果还没有保存，请保存jmx_examples.zip进入你的work_dir目录。
  2. 通过在终端窗口中使用以下命令来解压缩样本类包。 
    unzip jmx_examples.zip
  3. 从内部编译示例Java类work_dir目录。 
    javac com/example/*.java
  4. 开始Main应用程序，指定公开的属性Main用于远程管理。（对于Windows，请使用克拉（ ^ ）而不是反斜杠（ \ ）将长命令分成多行）：
    java -Dcom.sun.management.jmxremote.port=9999 \
     -Dcom.sun.management.jmxremote.authenticate=false \
     -Dcom.sun.management.jmxremote.ssl=false \
     com.example.Main

    确认Main等待发生的事情。

  5. 在另一台计算机上的另一个终端窗口中启动JConsole：
    jconsole

    将显示“新建连接”对话框，其中显示了可以在本地连接的正在运行的JMX代理的列表。

  6. 选择“远程进程”，然后在“远程进程”字段中键入以下内容：
    hostname:9999

    在这个地址hostname是在其上的远程计算机的名称Main应用程序正在运行，并且9999是现成的JMX连接器将连接到的端口号。

  7. 单击连接。

    Java虚拟机（Java VM）当前活动的摘要，其中Main显示正在运行。

  8. 单击“ MBeans”选项卡。

    该面板显示了当前在远程MBean服务器中注册的所有MBean。

  9. 在左侧框架中，展开com.example MBean树中的节点。

    您看到示例MBean Hello由创建和注册的Main 。如果单击Hello ，即使MBean树在另一台计算机上运行，也会在MBean树中看到其关联的“属性和操作”节点。

  10. 要关闭JConsole，请选择连接->退出。
