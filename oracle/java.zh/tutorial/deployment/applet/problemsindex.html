解决小程序常见问题

本节介绍编写Java applet时可能遇到的一些常见问题。在每个问题之后列出了可能的原因和解决方案。

问题：我的小程序没有显示。

  • 检查Java控制台日志中是否有错误。
  • 检查小程序的Java网络启动协议（JNLP）文件的语法。错误的JNLP文件是没有明显错误的失败的最常见原因。
  • 如果使用进行部署，请检查JavaScript语法runApplet部署工具包的功能。有关详细信息，请参见部署Applet 。

问题： Java控制台日志显示java.lang。ClassNotFoundException。

  • 确保正确编译了Java源文件。
  • 如果使用标记，请检查是否已在中正确指定了applet JAR文件的路径archive属性。
  • 如果使用JNLP文件启动，请检查jar JNLP文件中的标签。
  • 确保小程序的JAR文件，JNLP文件和网页位于正确的目录中，并且彼此准确引用。

    • 问题：我曾经能够构建一次代码，但是即使没有编译错误，现在构建也会失败。

    • 关闭浏览器，然后再次运行构建。浏览器很可能在JAR文件上具有锁定，因此构建过程无法重新生成JAR文件。

    问题：当我尝试加载包含小程序的网页时，浏览器将我重定向到www.java.com没有任何警告

    • 网页上的applet最有可能是使用Deployment Toolkit脚本进行部署的。该小程序可能需要比客户机上当前存在的Java Runtime Environment软件更高的版本。检查minimumVersion的参数runApplet小程序网页中的功能。有关详细信息，请参见部署Applet 。

    问题：我修复了一些错误，并重新构建了applet的源代码。重新加载小程序的网页时，未显示我的修复程序。

    • 您可能正在查看小程序的先前缓存版本。关闭浏览器。打开Java控制面板并删除Internet临时文件。这将从缓存中删除您的applet。尝试再次查看小程序。
