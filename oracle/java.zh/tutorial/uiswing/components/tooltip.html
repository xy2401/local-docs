如何使用工具提示

为任何人创建工具提示JComponent对象很容易。使用setToolTipText为组件设置工具提示的方法。例如，要将工具提示添加到三个按钮，只需添加三行代码：

b1.setToolTipText("Click this button to disable the middle button.");
b2.setToolTipText("This middle button does not react when you click it.");
b3.setToolTipText("Click this button to enable the middle button.");

当程序的用户将光标停在程序的任何按钮上时，该按钮的工具提示就会出现。您可以通过运行ButtonDemo示例， 如何使用按钮，复选框和单选按钮中对此进行了说明。这是当光标停留在鼠标左键时出现的工具提示图片。 ButtonDemo例。

对于具有多个部分的组件（例如选项卡式窗格），更改工具提示文本以反映光标下方的组件部分通常很有意义。例如，选项卡式窗格可以使用此功能来解释单击光标下的选项卡时将发生的情况。实施选项卡式窗格时，可以在传递给addTab要么setToolTipTextAt方法。

即使在没有用于设置零件特定工具提示文本的API的组件中，您通常也可以自己完成这项工作。如果组件支持渲染器，则可以在自定义渲染器上设置工具提示文本。表格和树部分提供了由自定义渲染器确定的工具提示文本的示例。适用于所有人的替代方案JComponent s正在创建组件的子类并覆盖其子类getToolTipText(MouseEvent)方法。

工具提示API

设置工具提示所需的大多数API都属于JComponent类，因此被大多数Swing组件继承。可以在各个类中找到更多的工具提示API，例如JTabbedPane 。通常，这些API足以指定和显示工具提示。您通常不需要直接处理实现类JToolTip和ToolTipManager 。

下表列出了工具提示中的API JComponent类。有关各个组件对工具提示的支持的信息，请参阅所涉及组件的“操作方法”部分。

工具提示中的API JComponent类
方法 目的
setToolTipText（String） 如果指定的字符串不为null，则此方法将组件注册为具有工具提示，并在显示时为工具提示提供指定的文本。如果参数为null，则此方法将关闭该组件的工具提示。
字符串getToolTipText（） 返回先前使用指定的字符串setToolTipText 。
字符串getToolTipText（MouseEvent） 默认情况下，返回相同的值getToolTipText() 。多部分组件，例如JTabbedPane ， JTable和JTree重写此方法以返回与鼠标事件位置关联的字符串。例如，选项卡式窗格中的每个选项卡可以具有不同的工具提示文本。
点getToolTipLocation（MouseEvent） 返回（在接收组件的坐标系统中）出现在组件工具提示左上角的位置。参数是导致工具提示显示的事件。默认返回值为null，它告诉Swing系统选择一个位置。
