SortedMap接口

一种SortedMap是一个Map根据键的自然顺序或以Comparator在提供的时间SortedMap创建。自然排序和Comparator在“ 对象排序”部分中进行了讨论。的SortedMap界面提供正常的操作Map操作以及以下内容：

  • Range view —在排序的地图上执行任意范围的操作
  • Endpoints —返回排序后的映射中的第一个或最后一个键
  • Comparator access —返回Comparator （如果有的话），用于对地图进行排序

以下界面是Map类似物SortedSet 。

public interface SortedMap<K, V> extends Map<K, V>{
    Comparator<? super K> comparator();
    SortedMap<K, V> subMap(K fromKey, K toKey);
    SortedMap<K, V> headMap(K toKey);
    SortedMap<K, V> tailMap(K fromKey);
    K firstKey();
    K lastKey();
}

地图操作

行动SortedMap继承自Map在排序图和法线图上的行为相同，但有两个例外：

  • 的Iterator由...返回iterator对任何已排序地图的操作Collection视图按顺序遍历集合。
  • 由返回的数组Collection意见” toArray操作按顺序包含键，值或条目。

尽管接口无法保证，但toString的方法Collection所有Java平台的视图SortedMap实现按顺序返回包含视图的所有元素的字符串。

标准构造

按照惯例，所有通用Map实现提供了一个采用Map ; SortedMap实现也不例外。在TreeMap ，此构造方法创建一个实例，该实例根据其键的自然顺序对其条目进行排序。这可能是一个错误。最好动态检查以查看是否指定了Map实例是一个SortedMap如果是，则根据相同标准（比较器或自然排序）对新地图进行排序。因为TreeMap采取的方法，还提供了一个构造函数SortedMap并返回一个新的TreeMap包含与给定相同的映射SortedMap ，根据相同标准排序。请注意，参数的编译时类型（而不是其运行时类型）决定了是否SortedMap优先于普通调用构造函数map构造函数。

SortedMap根据惯例，实现还提供了一个Comparator并返回根据指定的顺序排序的空地图Comparator 。如果null传递给此构造函数，它返回一个Map根据其键的自然顺序对映射进行排序。

与SortedSet的比较

因为这个界面是精确的Map类似物SortedSet ，则SortedSet接口部分中的所有惯用法和代码示例均适用于SortedMap仅进行一些细微的修改。
